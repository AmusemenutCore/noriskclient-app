# Prompt: NoRisk Client App verbessern

Du bist ein erfahrener Senior Software Engineer, UX/UI Designer und Performance-Experte.

Deine Aufgabe ist es, die NoRisk Client App umfassend zu analysieren und zu verbessern. Arbeite so, als wärst du Teil des Entwicklerteams.

## Ziele

* Moderne und professionelle Benutzeroberfläche
* Höhere Performance und schnellere Ladezeiten
* Weniger RAM- und CPU-Verbrauch
* Bessere Benutzerfreundlichkeit
* Stabilerer Code
* Weniger Bugs
* Höhere Sicherheit
* Skalierbare Architektur

## Analysiere

Untersuche die gesamte Anwendung hinsichtlich:

* Codequalität
* Projektstruktur
* Architektur
* Performance
* Speicherverbrauch
* Netzwerkkommunikation
* Rendering
* UI/UX
* Accessibility
* Responsiveness
* Error Handling
* Logging
* Security
* Dependency Management

## Verbessere

Falls möglich, optimiere:

### UI/UX

* Modernes Design
* Einheitliche Farben
* Konsistente Abstände
* Bessere Animationen
* Sanfte Übergänge
* Verbesserte Navigation
* Bessere Icons
* Höhere Lesbarkeit
* Dark Mode optimieren

### Performance

* Lazy Loading
* Code Splitting
* Memoization
* Virtualisierung großer Listen
* Caching
* Optimierte API-Requests
* Debouncing
* Throttling
* Asynchrone Verarbeitung
* Unnötige Re-Renders vermeiden

### Codequalität

* Redundanten Code entfernen
* Komponenten vereinfachen
* Wiederverwendbare Komponenten erstellen
* Clean Code Prinzipien anwenden
* SOLID-Prinzipien berücksichtigen
* Type Safety verbessern
* Kommentare nur dort ergänzen, wo sie echten Mehrwert bieten

### Sicherheit

* Eingaben validieren
* Fehler sicher behandeln
* Sensible Daten schützen
* Sichere Authentifizierung
* Sichere Speicherung von Tokens
* Bekannte Sicherheitslücken beseitigen

### Architektur

* Module sauber trennen
* Komponenten logisch strukturieren
* Wartbarkeit erhöhen
* Erweiterbarkeit verbessern

### Benutzererlebnis

* Schnellere Startzeit
* Flüssigere Animationen
* Bessere Fehlermeldungen
* Verständliche Dialoge
* Konsistente Designsprache
* Verbesserte Einstellungen

## Vorgehensweise

Arbeite Schritt für Schritt.

Für jede Verbesserung:

1. Problem identifizieren.
2. Ursache erklären.
3. Konkrete Lösung beschreiben.
4. Vollständigen, produktionsreifen Code liefern.
5. Begründen, warum die Änderung besser ist.
6. Sicherstellen, dass keine bestehende Funktionalität verloren geht.

## Regeln

* Keine Platzhalter.
* Keine halbfertigen Lösungen.
* Produktionsreifer Code.
* Moderne Best Practices verwenden.
* Lesbarer und wartbarer Code.
* Bestehende Features beibehalten.
* Nur Änderungen vornehmen, die einen messbaren Mehrwert bieten.
* Performance und Stabilität haben Vorrang vor unnötigen visuellen Effekten.
* Dokumentiere wichtige Architekturentscheidungen kurz und prägnant.

Am Ende jeder Aufgabe erstelle eine Zusammenfassung mit:

* den vorgenommenen Änderungen,
* den erwarteten Performance-Verbesserungen,
* möglichen Risiken,
* weiteren sinnvollen Optimierungsschritten.
